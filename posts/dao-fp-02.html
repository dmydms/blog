Bartosz Milewski의 함수형 프로그래밍의 도 The Dao of Functional Programming 2장 “합성Composition”을 읽고 요약했습니다.

2.1. 합성Composition

프로그래밍에 있어서 가장 중요한 개념 중 하나는 합성composition입니다. 범주론에서 대상과 사상(화살표)을 다루는 이유도 마찬가지입니다.

대상 aaa에서 bbb로 향하는 사상 fff와, 대상 bbb에서 ccc로 향하는 사상 ggg가 있으면 이를 합성하여 aaa에서 ccc로 향하는 사상 hhh를 만들 수 있습니다.

a -> b [label=f]
b -> c [label=g]
a -> c [label=h, constraint=false]

이는 수학에서 함수의 합성과 같으며 이렇게 표기합니다.

h=g∘fh = g \circ fh=g∘f

해스켈에서는 이렇게 씁니다.

h = g . f

수학적 표기와 (대부분의) 프로그래밍 언어의 문법이 위 그래프와 반대인 이유는 함수의 인자를 함수 이름 뒤에 적는 관습 때문입니다. h=g(f(x))h = g(f(x))h=g(f(x)) 처럼요.

영어로는 ”hhh is equal to ggg after fff“라고 읽으면 자연스러운데, 우리말로 ggg와 fff의 순서를 바꾸지 않고 자연스럽게 읽을 방법은 아직 모르겠습니다.

만약 위 예시에서 ggg가 사실은 jjj와 kkk의 합성이라고 가정해보면 이렇게 쓸 수 있습니다.

h=(j∘k)∘fh = (j \circ k) \circ fh=(j∘k)∘f

범주론의 합성은 수학에서의 합성과 마찬가지로 결합법칙associativity이 성립하기 때문에 위 식은 아래처럼 쓸 수 있습니다.

h=j∘(k∘f)h = j \circ (k \circ f)h=j∘(k∘f)

따라서 괄호는 생략할 수 있습니다.

h=j∘k∘fh = j \circ k \circ fh=j∘k∘f

사상들morphisms 사이의 대응mappings인 전합성pre-composition과 후합성post-composition이 존재합니다.

사상 fff가 사상 hhh에 후합성되면 사상 f∘hf \circ hf∘h가 만들어집니다. fff에 의한 후합성을 (f∘−)(f \circ -)(f∘−)으로 표현합니다. f:a→bf: a \rightarrow bf:a→b 라면, aaa로 향하는 어떠한 사상도 모두 fff에 의해 후합성될 수 있습니다. 따라서 사상 f∘hf \circ hf∘h는 사상들 간의 대응 (f∘−)(f \circ -)(f∘−)를 만들내는데, 이 대응은 aaa를 향하는 사상들을 bbb로 향하는 사상들로 변환하는 역할을 합니다.

rankdir=TD;
a -> b [label="f", color="#33FF33"];
{x1 x2} -> a;
{y1 y2} -> b;

{rank = same; a; b;}
{rank = same; x1; x2; y1; y2}
x2 -> y1 [label="(f ∘ −)"; style="dashed", color="#33FF33"];

즉, 무언가로부터 aaa를 만들어내는 사상들(x1,x2,...x_1, x_2, ...x1​,x2​,...)을, 무언가로부터 bbb를 만들어내는 사상들(y1,y2,...y_1, y_2, ...y1​,y2​,...)로 바꿔줍니다.

해스캘 코드로 최대한 명시적으로 표현해보면 이렇습니다.

h :: Int -> Bool
h x = x /= 0

f :: Bool -> String
f False = "False"
f True = "True"

g :: Int -> String
g = f . h

h는 Int를 받아서 Bool을 반환하는 함수입니다. Bool을 받아서 String을 반환하는 함수 f를 h에 후합성한 g는 Int를 받아서 String을 반환하는 함수가 됩니다. 즉, Bool을 반환하는 임의의 함수에 f를 후합성하면 그 결과로 얻어지는 함수들은 String을 반환합니다.

후합성의 쌍대dual 개념은 전합성입니다. fff의 전합성은 (−∘f)(- \circ f)(−∘f)이며, aaa로부터 무언가를 만들어내는 사상들(x1,x2,...x_1, x_2, ...x1​,x2​,...)을 bbb로부터 무언가를 만들어내는 사상들(y1,y2,...y_1, y_2, ...y1​,y2​,...)로 바꿔줍니다.

rankdir=TD;
overlap = false;

a -> b [label="f", color="#33FF33"];
a -> {x1 x2};
b -> {y1 y2};

x2 -> y1 [label="(- ∘ f)"; style="dashed"; dir=back, color="#33FF33"];

{rank = same; a; b;}
{rank = same; x1; x2; y1; y2}

전합성과 후합성은 각각 (−∘−)(- \circ -)(−∘−)에 대한 부분 적용partial application으로 볼 수 있습니다.

프로그래밍에서 밖으로 향하는 화살표들은 소스로부터 데이터를 추출한다는 의미이며, 안으로 들어오는 화살표들은 무언가를 만들어낸다는 의미입니다. 나가는 화살표는 인터페이스, 들어오는 화살표는 생성자라고 말할 수 있습니다.

2.2. 함수 적용Function application

끝 대상terminal object에서 aaa로 향하는 사상 xxx를 aaa의 원소element로 볼 수 있습니다.

1→xa1 \overset{x}{\rightarrow} a1→xa

여기에 aaa에서 bbb로 향하는 사상 fff를 합성하여 사상 yyy를 얻을 수 있습니다.

1 -> a [label=x];
a -> b [label=f];
1 -> b [label=y, constraint=false];

이때 yyy는 bbb의 원소라고 할 수 있으며, fff는 aaa의 원소를 bbb의 원소로 대응시키는 역할을 합니다. 이걸 xxx를 함수 fff에 적용application한다고 부릅니다. 함수의 적용은 이렇게 표현합니다.

y=fxy = fxy=fx

일반적인 프로그래밍 언어, 예를 들어 자바스크립트에서는 이렇게 표현합니다.

const y = f(x)

2.3. 항등Identity

사상은 변화를 나타낸다고 볼 수도 있습니다. 대상 aaa가 대상 bbb로 바뀌니까요. aaa에서 aaa로 향하는 사상은 대상 그 자체의 변화를 나타냅니다. 하지만 변화에도 쌍대dual가 존재합니다. 변화가 없는 것, 즉 항등identity입니다.

범주category의 모든 대상은 특별한 사상으로 항등을 가집니다. 대상 aaa에 대한 항등 사상을 idaid_aida​로 표시합니다.

사상 f:a→bf: a \rightarrow bf:a→b가 있을 때 다음이 성립합니다.

idb∘f=f=f∘idaid_b \circ f = f = f \circ id_aidb​∘f=f=f∘ida​

그림으로는 이렇게 표현합니다.

a -> b [label="f"];
a -> a [label="id_a", color="#33FF33"];
b -> b [label="id_b", color="#33FF33"];

해스켈에서는 id 함수가 항등을 나타냅니다.

id x = x

논리학에서는 항등명제tautology라고 부릅니다. “a가 참이라면 a는 참이다”라는 의미입니다.
